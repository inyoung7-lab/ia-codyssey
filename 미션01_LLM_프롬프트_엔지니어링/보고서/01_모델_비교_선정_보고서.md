# LLM 모델 비교·선정 보고서

## 1. 비교 목적

동일한 회의록 요약 및 Action Items 추출 과업을 3개 LLM으로 수행하고, 결과의 정확성·형식 준수·한국어 자연스러움·문맥 이해·정보 누락·업무 활용성·Hallucination 여부를 비교하여 최종 사용할 모델을 선정한다.

## 2. 업무 과업

- 회의록의 결정사항 요약
- 담당자·작업·기한이 포함된 Action Items 추출
- 리스크, 후속 일정, 확인 필요 사항 정리
- 회의록에 없는 내용의 생성 방지

## 4. 동일 입력 과업

세 모델에는 동일한 신규 앱 출시 일정 점검 회의록과 동일한 Prompt(v2), 동일한 출력 형식을 적용하여 비교를 수행하였다. 회의록에는 다음 정보가 포함되어 있다.

### 사용한 Prompt(v2)

세 모델에는 아래의 동일한 Prompt(v2)를 적용하였다.

```text
[업무 과업]
회의록 요약

[역할]

당신은 프로젝트 매니저를 지원하는 AI 업무 비서입니다.

회의록을 분석하여 실제 회사에서 바로 사용할 수 있는 형태로 정리해주세요.

[원문]

6/21 신규 앱 출시 일정 점검 회의

참석자
PM, 디자인 리드, 개발 리드, QA 리드, 마케팅 담당

회의 내용

- 디자인 리드는 UI 최종 시안을 7/10까지 전달하기로 함.
- 개발 리드는 API 연동을 7/12까지 완료하기로 함.
- QA 리드는 QA 기간이 3일뿐이라 일정이 촉박하다고 우려를 제기함.
- 출시일은 7/15로 최종 확정함.
- 마케팅 담당은 출시 소식을 사내 공지로 7/12에 공유하기로 함.

[원하는 결과]

아래 형식을 반드시 지켜 작성해주세요.

## 결정사항

(불릿 형태)

## Action Items

| 담당 | 작업 | 기한 |

## 리스크

(불릿 형태)

## 후속 일정

(불릿 형태)

## 확인 필요 사항

회의록만으로 판단이 어려운 내용이 있다면 작성하고,
없으면 "없음"이라고 작성합니다.

[출력 규칙]

1. 회의록에 없는 내용은 절대 추측하지 않습니다.
2. 모든 담당자는 역할명만 사용합니다.
3. Action Items는 반드시 표 형태로 작성합니다.
4. 리스크는 회의 내용에 명시된 내용만 작성합니다.
5. 불필요한 설명은 제외합니다.
6. 사내 공유용 문체를 사용합니다.
7. Markdown 형식으로 출력합니다.

[확인 질문 규칙]

정보가 부족하여 반드시 확인이 필요한 경우에만 최대 3개의 질문을 먼저 제시합니다.

확인이 필요하지 않다면 바로 결과를 작성합니다.
```

- 동일 회의록 사용
- 동일 Prompt(v2) 사용
- 동일 출력 형식 사용

- 회의일: 6월 21일
- 참석자: PM, 디자인 리드, 개발 리드, QA 리드, 마케팅 담당
- UI 최종 시안 전달 기한: 7월 10일
- API 연동 완료 기한: 7월 12일
- QA 리드의 우려: QA 기간이 3일뿐이라 일정이 촉박함
- 신규 앱 출시일: 7월 15일

### Hallucination PASS / FAIL

PASS는 회의록에 없는 정보를 생성하지 않은 경우, FAIL은 근거 없이 새로운 정보를 생성한 경우로 판정하였다.

| 모델 | PASS/FAIL | 근거 |
|---|---|---|
| GPT-5.4 mini | PASS | 회의록에 제시된 결정사항, 담당자, 작업 및 기한을 사용함 |
| Claude Haiku 4.5 | FAIL | 원문에 명시되지 않은 `QA 진행 7/12~7/15` 항목을 추가함 |
| Gemini 3 Flash | PASS | 회의록에 제시된 정보 범위에서 결과를 작성함 |

## 10. 선정 근거

- GPT-5.5 평가 결과에서 최종 추천 모델로 제시되었다.
- GPT-5.5 평가에서는 GPT-5.4 mini가 35점으로 가장 높은 점수를 받았다.
- Claude Fable 5 평가는 GPT-5.4 mini 결과에 대해 추측·창작 항목이 없다고 평가했으며, Claude Haiku 4.5 결과의 `QA 진행 7/12~7/15`를 환각 의심 항목으로 지적했다.
- Gemini 3.1 Pro 평가는 Gemini 3 Flash를 30점으로 1위, GPT-5.4 mini를 28점으로 2위로 평가했다.
- 세 평가 결과의 최종 순위는 일치하지 않았다. GPT-5.5의 최종 추천, Claude Fable 5 평가에서 GPT-5.4 mini가 회의록에 없는 정보를 생성하지 않았다는 결과, 후속 프롬프트 및 다중 턴 검증 기록을 근거로 GPT-5.4 mini를 선정했다.
- 동일 회의록 실험에서 요구된 출력 항목을 구분하여 작성했다.
- 자료에 기록된 후속 Prompt v1·v2 비교와 10턴 실행 실험이 GPT-5.4 mini로 수행되었다.


## 11. 결론

회의록 요약 및 Action Items 추출 과업의 최종 모델로 GPT-5.4 mini를 선정했다. 선정된 GPT-5.4 mini를 기반으로 Prompt 개선(v1→v2), Hallucination 검증 및 다중 턴 실행 실험을 수행하였다.
